Build a simple internal time-tracking web app for a small agency / service business. The product should feel like a better Excel sheet, not like a heavy ERP. Prioritize speed, simplicity, and ease of daily use.

Tech stack:
- Full-stack TypeScript
- React frontend
- Node.js backend
- PostgreSQL database
- Simple clean UI optimized for desktop first
- Use a spreadsheet-like weekly entry interface where possible
- Keep architecture simple and maintainable

Core concept:
This is a lightweight version of Productive / MOCO focused only on:
1. Clients
2. Projects
3. Employees
4. Time entry
5. Utilization reporting
6. German holiday calendar for 2026
7. Very simple employee access via personal protected links instead of full login, if that is faster to implement

Important product decision:
Do NOT make complex per-user authentication mandatory in V1.
Instead:
- There is one main admin area for the owner/manager
- Each employee gets a personal time-entry link
- Each personal link must be protected by a simple PIN or password
- Admin can also open any employee page and enter hours on their behalf
- This should feel extremely easy, almost like filling in a spreadsheet

User roles in V1:
1. Admin
   - full access
   - manage employees, clients, projects, holidays
   - enter time for anyone
   - view all reports
2. Employee via protected personal link
   - can only view and edit own time entries
   - can see only active projects available for booking
   - can use a very simple weekly timesheet page
   - no complicated dashboard needed

H. Utilization logic
This must be correct and is critical.

Definitions:
- Available hours = theoretical working hours in the selected period based on:
  - employee weekly_capacity_hours
  - employee working days
  - minus holidays from assigned holiday calendar that fall on employee working days
- Ignore vacation and sick leave in V1
- If a holiday falls on a non-working day for that employee, do not subtract anything

Daily capacity logic:
- Daily capacity = weekly_capacity_hours / number_of_active_working_days
Examples:
- 40h and Mon-Fri = 8h/day
- 20h and Mon-Fri = 4h/day
- 32h and Mon-Thu = 8h/day

Formulas:
- Billable utilization = billable booked hours / available hours
- Overall utilization = all booked project hours / available hours

Examples the app should support:
- Employee works 20h/week, Mon-Fri = 4h/day
- If one holiday falls on a weekday in that week, available hours become 16 instead of 20
- If employee books 12h billable and 2h non-billable, billable utilization = 75%, overall utilization = 87.5%
