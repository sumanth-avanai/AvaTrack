We need to update the "Edit Booking" dialog in the Resource Planner 
to prevent over-booking by showing complete resource allocation 
visibility.

CURRENT ISSUE:
When allocating hours to an employee in the "Edit booking" section, 
the "Role Budget" only shows planned hours, not actual logged hours. 
This causes over-booking because we can't see what work is already 
in progress.

LOCATION:
Resource Planner → Edit Booking dialog → "Role Budget" section

NEW FIELDS TO DISPLAY:

Update the Role Budget box to show:

┌─────────────────────────────────┐
│ ROLE BUDGET                     │
├─────────────────────────────────┤
│ Budgeted:          100d         │
│ Already Planned:   -16.5d       │
│ Already Logged:    -20.0d       │
│ Available:         63.5d        │
│                                 │
│ This Booking:      +43d         │
├─────────────────────────────────┤
│ Remaining After:   20.5d        │
└─────────────────────────────────┘

FIELD DEFINITIONS:

1. Budgeted: Total allocated budget for this role on this project
2. Already Planned: Hours from other bookings already scheduled
3. Already Logged: Actual hours already tracked/worked (IN-PROGRESS)
4. Available: Budget left to allocate (Budgeted - Planned - Logged)
5. This Booking: Hours being added in this current allocation
6. Remaining After: Budget left after this booking 
                   (Budgeted - Planned - Logged - This Booking)

CALCULATION LOGIC:

Available = Budgeted - Already Planned - Already Logged

Remaining After = Budgeted - Already Planned - Already Logged - This Booking

KEY POINT:
Use LOGGED hours (not INVOICED) for calculations because:
- Logged hours = actual work committed/in-progress
- Invoiced hours = just billing status, doesn't affect availability
- Using logged prevents over-booking of resources

EXAMPLE:

Role Budget: 100 days
Already Planned (other bookings): 16.5 days
Already Logged (work in progress): 20 days
Available to allocate: 100 - 16.5 - 20 = 63.5 days

New Booking being made: 43 days

Remaining After: 100 - 16.5 - 20 - 43 = 20.5 days

WHY THIS MATTERS:
- Shows complete picture of what's planned AND what's actually being done
- Prevents allocating same person to multiple overlapping projects
- Clear visibility of true resource availability
- Prevents under/over-booking

Please update the Role Budget section in the Edit Booking dialog 
to display all four values (Budgeted, Already Planned, Already Logged, 
Available) plus the calculation based on LOGGED hours, not invoiced.
