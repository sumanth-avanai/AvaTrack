=== BUG 3: CSV uses wrong decimal separator ===

PROBLEM:
- German locale uses "," for decimals
- CSV should use "." for compatibility with Excel/tools
- Or use ";" as CSV separator if using ","

FIX:
- Use "." for decimals, "," as separator (international standard)

```javascript
function exportCSV(rows) {
  const csvContent = rows.map(row => 
    row.map(cell => {
      if (typeof cell === 'number') {
        return cell.toFixed(2); // Always use "." for decimals
      }
      // Escape strings with commas
      if (typeof cell === 'string' && cell.includes(',')) {
        return `"${cell}"`;
      }
      return cell;
    }).join(',')
  ).join('\n');
  
  download(csvContent, 'text/csv');
}
```
