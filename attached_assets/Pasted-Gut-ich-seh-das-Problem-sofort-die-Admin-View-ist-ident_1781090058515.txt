Hier der Replit-Prompt:

---

**Replit Prompt: Admin Timesheet List View**

Add a new Admin-only Timesheet view at `/admin/timesheets`. This is separate from the existing employee timesheet at `/timesheet` — do NOT modify the employee view.

**⚠️ IMPORTANT: Preserve ALL existing data and the employee timesheet view.**

**Requirements:**

**1. Route & Access**
- New route: `/admin/timesheets`  
- Only visible/accessible to admin users
- Add "Timesheets (Admin)" link to the sidebar under "Timesheet" for admins

**2. Filter Bar**
- Employee dropdown (default: All)
- Project dropdown (default: All)
- Month/Year picker (default: current month) — shows entries for that full month
- Optional: "Show only entries without role" toggle (to quickly find bad entries)

**3. Table View**
Columns: Date | Employee | Project | Role | Hours | Actions

- Sort by date descending by default
- Rows where Role is null/empty: highlight with amber background and warning icon ⚠️
- Show total hours at the bottom (filtered)
- Paginate if > 50 rows

**4. Inline Edit**
- Edit button (pencil icon) per row opens a small modal/inline form
- Editable fields: Project (dropdown), Role (dropdown filtered by project), Hours, Date
- Save updates the TimeEntry record
- Validation: hours must be > 0, project and role required

**5. Delete**
- Delete icon per row with confirmation ("Delete this entry?")
- Soft confirmation only — no bulk delete

**Acceptance Criteria:**
- [ ] Admin can view all time entries for any employee for a given month
- [ ] Admin can filter by employee and/or project
- [ ] Entries without a role are visually flagged
- [ ] Admin can edit project, role, hours, date of any entry
- [ ] Admin can delete an entry with confirmation
- [ ] Employee timesheet view (`/timesheet`) is completely unchanged
- [ ] No data loss
