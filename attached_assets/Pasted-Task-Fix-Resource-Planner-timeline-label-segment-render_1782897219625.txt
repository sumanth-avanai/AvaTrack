Task: Fix Resource Planner timeline label/segment rendering (regression from
previous change)

CONTEXT: The previous change introduced day-level booking rendering, but the
implementation renders a separate label inside every single day cell, which
becomes unreadable at Week/Month/Quarter zoom (labels are truncated to a
handful of characters, e.g. only 2-4px of text visible per day). This must
be fixed without reverting the underlying day-level availability logic
(0h days should still break a segment, proportional fill logic must stay).

IMPORTANT: Do not modify any Booking, Employee, Project, Holiday, or
Vacation data. This is a rendering-only fix.

FIX 1 — Label rendered once per segment, not once per day
- A "segment" = a contiguous run of days with >0 booked hours for the same
  booking (same role/project), as already defined in the previous task.
- The role name label renders exactly ONCE per segment, not repeated in
  every day-cell within that segment.
- Position the label so it stays visible even when the segment is
  horizontally scrolled - e.g. sticky-positioned at the visible left edge of
  the segment's rendered area (standard Gantt-bar label behavior), rather
  than always at the segment's absolute start.
- If a segment is narrower than the space needed to render the label at
  minimum readable size (see Fix 3), fall back to showing nothing in the
  bar and rely on the existing hover/click tooltip for the role name -
  do NOT shrink text below the minimum or truncate to unreadable fragments.

FIX 2 — Day-level proportional fill only at Week/Day zoom
- The per-day proportional fill background + "Xh free" / "-Xh" overlay
  (introduced in the previous task) should only render when the zoom level
  is Week or Day AND the resulting day-cell width is at least ~60px.
- At Month and Quarter zoom, render each segment as a single continuous,
  evenly colored bar across its date range (still broken/split only at
  genuine 0h days, per the existing segment definition), WITHOUT the
  per-day fill/label overlay - i.e. revert to the simpler continuous-bar
  rendering for these zoom levels, keeping only: project color, one role
  label per segment (Fix 1), and segment breaks at 0h days.

FIX 3 — Minimum font size
- No text in the timeline (labels, "Xh free", overbooking values) may render
  below 11px under any zoom level. If content does not fit at 11px, hide it
  and rely on tooltip/click detail instead of shrinking further.

ACCEPTANCE CRITERIA
- [ ] In Week zoom, a booking with e.g. Mon 8h / Tue 4h / Wed-Fri 0h still
      shows the day-level proportional fill and "4h free" on Tuesday, exactly
      as before, but the role name appears once for the Mon-Tue segment, not
      repeated separately in the Monday cell and the Tuesday cell.
- [ ] In Month and Quarter zoom, each booking segment renders as one
      continuous colored bar with exactly one readable role-name label,
      no per-day subdivision or repeated text.
- [ ] No label text renders below 11px anywhere in the timeline.
- [ ] Existing Bookings/Employees/Projects/Holidays/Vacations remain
      unchanged in the database.
- [ ] Absence icons (star/sun/thermometer/X) still render correctly and are
      unaffected by this fix.
