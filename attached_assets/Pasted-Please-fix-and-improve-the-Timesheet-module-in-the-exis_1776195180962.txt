2. FEATURE: NON-BOOKABLE DAYS SHOULD BE DISABLED / GREYED OUT
==================================================

Add clear visual and functional handling for non-bookable days in the weekly timesheet.

Non-bookable days include:
- public holidays from the employee’s assigned holiday calendar
- employee non-working days based on working_days_mask
- days before contract_start_date
- days after contract_end_date (if present)
- vacation / absence days for that employee

Expected behavior:
- Non-bookable cells must be visibly greyed out
- Non-bookable cells must not allow hour entry
- If a user clicks into a non-bookable cell, prevent editing
- Optionally show a tooltip or helper text explaining why the day is unavailable

Priority:
- clear visual distinction
- no accidental booking on unavailable days

Required logic:
For each cell in the weekly timesheet, determine whether booking is allowed for that employee and date.

A cell is bookable only if:
- date is within employee contract period
- date is one of the employee working days
- date is not a holiday in assigned holiday calendar
- date is not covered by employee vacation/absence

If any of the above is false:
- cell is disabled
- cell is greyed out
- no hours can be entered

Also:
- weekly totals should ignore disabled cells
- no placeholder editable input in disabled cells

3. FEATURE: RECURRING / MULTI-WEEK TIME ENTRY
==================================================

Add a fast way to enter repeated hours across multiple weeks without filling every single day manually.

Goal:
If someone worked on the same project for multiple weeks, they should be able to apply a repeated booking pattern across a date range.

Add a new timesheet action such as:
- "Repeat entry"
OR
- "Recurring booking"
OR
- "Apply to multiple weeks"

Behavior:
User can choose:
- employee
- project
- start date
- end date
- weekdays to apply (Mon, Tue, Wed, Thu, Fri, Sat, Sun)
- hours per selected day
- optional note

Then the system creates time entries for all matching bookable days in that range.

Required rules:
- only create entries on selected weekdays
- skip non-bookable days automatically:
  - holidays
  - non-working days
  - vacation days
  - before contract start
  - after contract end
- do not create invalid entries on disabled days

Conflict handling:
If an entry already exists for the same employee + project + date:
- choose one clear MVP behavior and implement it consistently
Preferred MVP behavior:
- add a confirmation modal with:
  - skip existing entries
  - overwrite existing entries
Use the simplest maintainable version if modal is too heavy, but behavior must be explicit and consistent.

UI suggestion:
Option A:
- add a "Repeat booking" button above the weekly timesheet

Option B:
- add action menu on a project row

Use whichever integrates best with current timesheet UI.
