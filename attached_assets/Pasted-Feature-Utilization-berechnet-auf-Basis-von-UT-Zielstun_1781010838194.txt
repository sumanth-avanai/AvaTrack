Feature: Utilization % berechnet auf Basis von UT-Zielstunden
Context
Employees already have a utilizationTarget field (0–100, nullable) set via the Employee module. This prompt changes how Utilization % is calculated in the Reports module. Currently util % = logged ÷ workingTargetHours. The new formula uses a personal UT target hours value as the denominator.

⚠️ IMPORTANT: Preserve ALL existing data. Schema changes must be additive and backward-compatible. Only calculation logic in the Reports module changes.

New Calculation Logic
For employees with utilizationTarget > 0
utTargetHours = workingTargetHours × (utilizationTarget / 100)
utilization%  = loggedHours ÷ utTargetHours
Example: Regina works 20h/week → workingTargetHours = 86.7h, utilizationTarget = 50 → utTargetHours = 43.3h → If she logs 43.3h → utilization = 100% (correct: all billable hours are booked)

Values above 100% are valid and must be displayed as-is (no capping).

For employees with utilizationTarget = 0 AND loggedHours > 0
These are freelancers or pass-through employees. They always count as 100% utilized when they have any logged hours.

For the team-level calculation, treat their utTargetHours = loggedHours (so they contribute exactly 100% to the weighted average).

Individual display: show 100% in the UTIL % column.

For employees with utilizationTarget = 0 AND loggedHours = 0
Exclude entirely from all calculations (numerator and denominator). Do not display a util % for them.

For employees with utilizationTarget = null (no target set)
Keep existing behavior: util % = loggedHours ÷ workingTargetHours (unchanged). These employees are included in the team total using their workingTargetHours as denominator.

Team-Level Util % (KPI card + Total row)
teamUtilization% = sum(loggedHours for included employees)
                 ÷ sum(utTargetHours for included employees)
Included employees: - utilizationTarget > 0 → always included, utTargetHours = workingTargetHours × (target/100) - utilizationTarget = 0 + loggedHours > 0 → included, utTargetHours = loggedHours - utilizationTarget = 0 + loggedHours = 0 → excluded - utilizationTarget = null → included, utTargetHours = workingTargetHours

The same team-level denominator change applies to the Allocation % KPI card: use the same utTargetHours per employee as denominator for planned hours too.

What does NOT change
The workingTargetHours value itself (173.3h base × hoursPerWeek/40) — still displayed in the TARGET column unchanged.
Allocation % formula and display — only the denominator in the KPI card aggregate changes (see above).
The color-coding logic from the previous feature (green ≥ target, red < target) — keep exactly as-is, just apply it to the new util % values.
No schema migrations needed — utilizationTarget field already exists.
Acceptance Criteria
[ ] Individual UTIL % for employees with utilizationTarget > 0 uses loggedHours ÷ (workingTargetHours × target/100) as formula.
[ ] Employees with utilizationTarget = 0 and logged hours show UTIL % = 100%.
[ ] Employees with utilizationTarget = 0 and no logged hours show no util % and are excluded from team totals.
[ ] Values above 100% are displayed as-is (e.g. 132%) — no capping.
[ ] Team-level Utilization % KPI card uses the new weighted denominator.
[ ] Team-level Allocation % KPI card uses the same weighted denominator.
[ ] Employees with no target set (null) behave exactly as before.
[ ] Existing data is fully preserved — no data loss.
