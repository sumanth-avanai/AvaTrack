# Replit Prompt: Project Status View – Block 1 Improvements

## Goal
Improve the Project Status view so it can be used effectively in senior leadership meetings. The view should give a quick, visual overview of all active projects — sorted, grouped, and enriched with relevant data — with minimal manual effort from the PM.

⚠️ IMPORTANT: Preserve ALL existing data and functionality. Only add or modify what is described below.

---

## Changes Required

### 1. Group projects: Active vs. Completed
- Split the project list into two groups: **Active** (any status that is not "Completed") and **Completed**
- Active projects are shown at the top, expanded by default
- Completed projects are shown below, **collapsed by default** with a toggle to expand (e.g. "Show 5 completed projects")
- Within each group, sort first by **Customer name (A–Z)**, then within the same customer by **Risk Level** (High → Medium → Low)

### 2. Risk Level row highlighting
- Rows with Risk Level **High** get a subtle red background tint
- Rows with Risk Level **Medium** get a subtle amber/yellow background tint
- Rows with Risk Level **Low** stay white/default
- This should work both in the list and in any detail view

### 3. PM field: Employee dropdown instead of free text
- In the **Project edit/create form**, replace the PM free-text input with a dropdown that loads from the existing Employees table
- The selected employee's name is stored as the PM for that project
- The **"All PMs" filter** in the Project Status view should continue to work and now filter by the selected employee values
- If an existing project already has a PM as free text that doesn't match an employee, show it as-is but allow the user to re-select from the dropdown to fix it

### 4. New field: Client Satisfaction
- Add a **Client Satisfaction** field to the project status update dialog (alongside General Status, Budget Status, Risk Level)
- Options: **Happy** / **Neutral** / **Critical**
- Display it in the Project Status list as a new column (after Risk Level), using simple color-coded badges:
  - Happy → green
  - Neutral → grey
  - Critical → red
- Also show it on the project detail page under "Current Health"
- This field is optional (can be left empty, shown as "—" if not set)

### 5. Budget Status: Automatic calculation (replace manual dropdown)
- Remove the manual **Budget Status** dropdown from the status update dialog
- Instead, calculate budget consumption automatically per project and display it in the Project Status list
- **Formula:**
  - Total Budget = sum of (budgetedDays × dayrate) across all ProjectRoles for this project
  - Budget Consumed = sum of (TimeEntry hours / 8 × dayrate) for all TimeEntries linked to roles in this project
  - Budget % = Budget Consumed / Total Budget × 100
- Display in the list as a percentage with a small progress bar, e.g. `▓▓▓▓░░ 64%`
- Color coding:
  - 0–74%: green
  - 75–89%: amber
  - ≥90%: red
- If no budget data exists for a project (no roles with dayrate/budgetedDays), show "—"
- On the project detail page, also show the absolute values: e.g. "€28,000 of €50,000 consumed (56%)"
- The existing "Budget Status" column in the list is replaced by this new calculated column

---

## Acceptance Criteria
- [ ] Active and completed projects are visually separated; completed are collapsed by default
- [ ] Within active projects, rows are sorted by customer A–Z, then High/Medium/Low risk
- [ ] High-risk rows have a red tint, medium rows amber tint, visible at a glance
- [ ] PM field in project form is a dropdown populated from Employees
- [ ] All PMs filter in Project Status works correctly with employee-based PMs
- [ ] Client Satisfaction field appears in status update dialog and as a column in the list
- [ ] Budget column shows auto-calculated % with progress bar and color coding
- [ ] Projects with no budget data show "—" in the budget column
- [ ] All existing data (status history, comments, etc.) is preserved
- [ ] No console errors, no broken routes
