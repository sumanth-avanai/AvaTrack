## Change 1: New color coding logic for ALLOC % and UTIL %

Replace the current binary (above/below target) color logic with a three-tier
relative threshold based on how close actual is to target.

### Formula

```
ratio = actual% / target%
```

| ratio | color |
|-------|-------|
| ≥ 0.75 (within 25% of target, or above) | 🟢 green |
| ≥ 0.50 and < 0.75 (25–50% below target) | 🟠 orange |
| < 0.50 (more than 50% below target) | 🔴 red |
| no target set (utilizationTarget = 0) | no color, show `—` |

This logic applies to:
- ALLOC % column (individual rows and Total row)
- UTIL % column (individual rows and Total row)
- Allocation KPI card (top)
- Utilization KPI card (top)

For the KPI cards, use the team-level actual vs. team goal (63%) as the ratio.

## Change 2: Allow sorting by Name column

The Name column header should be clickable and sort alphabetically, just like
the other columns already support sorting.

### Behavior

- Click "NAME" header → sort A→Z
- Click again → sort Z→A
- Click again → reset to default order (sorted by Util % descending)
- Show sort arrow (↑ / ↓) on the active column header

### Critical: "No utilization target" group always stays at the bottom

When sorting by name (or any other column), the group of employees with
`utilizationTarget = 0` must always remain pinned below the main group,
regardless of sort direction.

Example — sorting by Name A→Z:
```
Aemal Sayer
Keshav Reddy
Midhun Jose
Regina Strobl
Rohan Dhanawade
Sascha Seegebarth
Shruti Saxena
Willi Wendt
─────────────────────
No utilization target     ← always here, sorted A→Z within group
Christian Voigt
Punit Kumar
Veekshith Sumanth Gurram
─────────────────────
Total
```

Within the "No utilization target" group, names are also sorted
alphabetically when name sort is active.
