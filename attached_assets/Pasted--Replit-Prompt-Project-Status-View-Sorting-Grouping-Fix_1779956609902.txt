# Replit Prompt: Project Status View – Sorting & Grouping Fix

## Goal
Improve the visual grouping and sorting in the Project Status list so it's immediately clear which projects belong to which customer, and urgent projects are always visible first.

⚠️ IMPORTANT: Preserve ALL existing data and functionality.

---

## Changes Required

### 1. Customer as first column + visual grouping
- Move **Customer** to be the **first column** in the table (before Project)
- Group all projects by customer visually using a **group header row**:
  - The customer name appears as a subtle full-width (or spanning) header row above its projects
  - The customer column itself can be hidden or merged within the group — no need to repeat the customer name on every row
  - Use a light grey background or slightly bolder text for the group header to separate it visually from project rows
- Example layout:
  ```
  ▼ ACME CORP
       Website Redesign   |  In Progress  |  High  | ...
       Support & Maint.   |  —            |  Low   | ...
  ▼ BRAVO STUDIOS
       Brand Identity     |  In Progress  |  Medium| ...
  ```

### 2. Sorting order
- Sort customer groups by **highest risk project within that customer** (a customer with a High-risk project appears before a customer with only Low-risk projects)
- Within each customer group, sort projects by **Risk Level**: High → Medium → Low → no status
- Secondary sort within same risk level: by Project name A–Z

### 3. Collapsed completed projects
- Completed projects are still grouped under their customer but within the "Completed" section at the bottom
- The completed section remains collapsed by default as previously implemented

---

## Acceptance Criteria
- [ ] Customer is the first visual element in the table, shown as a group header
- [ ] Projects are visually grouped under their customer — no repeated customer name per row
- [ ] Customer groups are sorted so highest-risk customers appear first
- [ ] Within each customer group, projects are sorted High → Medium → Low risk
- [ ] Active and completed separation still works
- [ ] All filters (search, customer dropdown, PM dropdown) still work correctly with the new layout
